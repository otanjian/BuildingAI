# BuildingAI 使用说明

本文档说明如何在本地使用 BuildingAI，并通过 **MCP** 接入 **SAP ERP S/4**、**ERPNext** 等企业系统，在 AI 对话中调用外部工具。

- 首次安装环境（Node、PostgreSQL、Redis 等）请参阅：[安装说明文档.md](./安装说明文档.md)
- SAP 集成技术细节请参阅：[integrations/sap-abap-adt-mcp/README.md](./integrations/sap-abap-adt-mcp/README.md)

---

## 一、整体架构

```text
浏览器 (4091)
    ↓
BuildingAI API (4090)  ←→  PostgreSQL / Redis
    ↓ MCP (streamable-http)
    ├─ ERPNext MCP     http://127.0.0.1:8000/...   （需单独启动 ERPNext）
    └─ SAP ABAP ADT    http://127.0.0.1:8100/mcp  （由 start.sh 启动）
            ↓ supergateway
            ↓ yan252/mcp-abap-abap-adt-api (stdio)
            ↓ SAP ADT HTTPS
```

| 组件 | 默认端口 | 由谁启动 |
|------|----------|----------|
| 前端 | 4091 | `./start.sh` / `pnpm dev` |
| API | 4090 | `./start.sh` / `pnpm dev` |
| SAP MCP | 8100 | `./start.sh`（存在 SAP `.env` 时自动） |
| ERPNext MCP | 8000 | 需自行启动 ERPNext / Frappe 服务 |

## 二、快速开始

### 1. 准备主项目环境

```bash
cd buildingAI
cp .env.example .env
pnpm sync-env
# 按 安装说明文档.md 配置数据库、Redis 后：
pnpm install
```

### 2. 一键启动（推荐）

```bash
./start.sh restart
```

将启动：

- BuildingAI 开发栈（API + 前端）
- SAP MCP（若已配置 `integrations/sap-abap-adt-mcp/.env`）

常用命令：

| 命令 | 说明 |
|------|------|
| `./start.sh` | 启动（端口占用时会提示） |
| `./start.sh restart` | 重启全部托管服务 |
| `./start.sh stop` | 停止 |
| `./start.sh status` | 查看端口与进程 |
| `./start.sh logs sap` | 查看 SAP MCP 日志 |
| `./start.sh restart sap` | 仅重启 SAP MCP |
| `./start.sh -d restart` | 后台运行 dev（日志在 `.run/dev.log`） |

访问地址：

| 服务 | URL |
|------|-----|
| 前端 | http://localhost:4091 |
| API | http://localhost:4090 |
| 首次安装向导 | http://localhost:4090/install |

### 3. 根目录 `.env` 可选项

```ini
# 是否在 start.sh 时启动 SAP MCP（auto = 有 SAP .env 则启动）
START_SAP_MCP=auto

# 是否用 Docker 启动 redis + postgres
START_DOCKER_INFRA=false

# SAP MCP 端口
MCP_PORT=8100
```

## 三、接入 SAP ERP S/4（ABAP ADT MCP）

通过 [yan252/mcp-abap-abap-adt-api](https://github.com/yan252/mcp-abap-abap-adt-api) 连接 SAP，使用 **ADT HTTPS**，无需 NW RFC SDK / Docker。

### 1. 配置 SAP 连接

```bash
cp integrations/sap-abap-adt-mcp/.env.example integrations/sap-abap-adt-mcp/.env
```

编辑 `integrations/sap-abap-adt-mcp/.env`：

| 变量 | 说明 | 示例 |
|------|------|------|
| `SAP_URL` | ADT HTTPS 地址（向 basis 确认端口） | `https://sap.example.com:44300` |
| `SAP_USER` / `SAP_PASSWORD` | SAP 用户密码 | — |
| `SAP_CLIENT` | 客户端号 | `200` |
| `SAP_LANGUAGE` | 登录语言 | `ZH` |
| `SAP_SESSION_TYPE` | `stateless`（ECC 推荐）或 `stateful` | `stateless` |

> **注意**：`SAP_URL` 是 **ICM HTTPS + ADT** 地址，不是 SAProuter 字符串（如 `/H/host/S/3200`）。

### 2. 启动 SAP MCP

```bash
./start.sh restart sap
# 或一并启动：./start.sh restart
```

首次运行会自动 `git clone` 上游项目到 `vendor/`（已 gitignore），并 `npm install && npm run build`，可能需要数分钟。

成功标志：

```bash
./start.sh status
# :8100  SAP MCP   listening
```

本地探测（应能列出约 100+ 工具）：

```bash
curl --noproxy '*' -sS http://127.0.0.1:8100/mcp  # 有响应即可
```

### 3. 在控制台注册 MCP

1. 打开 http://localhost:4091/console/mcp  
2. **手动创建** 或 **从 JSON 导入**（可参考根目录 `mcp.json` 中的 `SAP-ABAP-ADT`）  
3. 填写：

| 字段 | 值 |
|------|-----|
| 名称 | `sap-abap-adt-mcp`（自定） |
| 通信方式 | **Streamable HTTP** |
| URL | `http://127.0.0.1:8100/mcp` |

4. 保存后点击 **检测连接**，状态应为 **可连通**，工具数量约 128。

### 4. SAP 能力说明

适合：ABAP 对象检索、源码、语法检查、传输、DDIC、`tableContents`、`runQuery` 等。

不适合：任意自定义 RFC/BAPI（除非通过 ADT 暴露）。与旧版 PyRFC 直连方案不同。

## 四、接入 ERPNext

ERPNext 通过 **Frappe Assistant Core** 暴露 MCP 端点，BuildingAI 以 HTTP 客户端连接。

### 1. 启动 ERPNext

确保本机 ERPNext（或 Frappe）已运行，且 MCP 端点可访问，例如：

```text
http://127.0.0.1:8000/api/method/frappe_assistant_core.api.fac_endpoint.handle_mcp
```

（端口与路径以你实际部署为准。）

### 2. 在控制台注册 MCP

| 字段 | 值 |
|------|-----|
| 名称 | `ERPnext-local` |
| 通信方式 | **Streamable HTTP** |
| URL | 上述 ERPNext MCP 地址 |
| 请求头 | `Authorization: token <api_key>:<api_secret>` |

可从 ERPNext 后台生成 API Key，**勿将真实 token 提交到 Git**。

根目录 `mcp.json` 仅作本地/Cursor 参考模板。

### 3. 检测连接

控制台 **检测连接** 通过后，应显示约 20+ 工具（视 ERPNext 插件配置而定）。

## 五、在 AI 对话中使用 MCP

控制台显示「已启用、可连通」≠ 聊天里一定可见。

### 控制台 vs 聊天

| 位置 | 作用 |
|------|------|
| **控制台 → MCP** | 管理员配置服务、检测连通、同步工具列表 |
| **聊天 → 工具** | 当前用户选择要启用的 **系统 MCP** |

### 在聊天中启用系统 MCP

1. 打开前端对话页 http://localhost:4091  
2. 输入框旁点击 **工具**（或 `+` 进入设置）  
3. 打开 **设置 → 工具 → 系统 MCP**  
4. 将 `ERPnext-local`、`sap-abap-adt-mcp` 等开关 **打开**  
5. 回到对话，在 **工具** 列表中勾选要用的 MCP（可多个）

此时应看到类似：**全部(2)**、**系统(2)**。

### 对话示例

- ERPNext：「查询物料主数据 …」「列出销售订单 …」  
- SAP：「用 searchObject 查找表 ZXXX」「tableContents 查看 MARA …」

具体能力取决于各 MCP 注册的工具；可在控制台 **查看工具** 浏览名称与说明。

## 六、Cursor 本地 MCP（可选）

根目录 `mcp.json` 已预置模板：

```json
{
  "mcpServers": {
    "ERPnext-local": { "type": "streamable-http", "url": "..." },
    "SAP-ABAP-ADT": { "type": "streamable-http", "url": "http://127.0.0.1:8100/mcp" }
  }
}
```

在 Cursor 中配置 MCP 前，请先 `./start.sh restart sap` 确保 `:8100` 在监听。

## 七、常见问题

### 1. 控制台可连通，聊天里只看到一个 MCP

**原因**：系统 MCP 需在个人 **设置 → 工具 → 系统 MCP** 中单独打开。  
**处理**：为缺失的服务打开开关后刷新对话页。

### 2. SAP MCP「不可连通」或工具数为 0

| 检查项 | 处理 |
|--------|------|
| `:8100` 未监听 | `./start.sh restart sap` |
| 启动脚本语法/克隆失败 | 查看 `.run/sap-mcp.log` |
| Git/npm 走失效代理 | `git config --global --unset http.proxy`，取消 `http_proxy` 后重试 |
| SAP 账号无 ADT 权限 | 联系 basis 开通 ADT |
| `SAP_URL` 错误 | 确认 HTTPS 端口（常见 `44300`） |

### 3. `PM2_LOG_DATE_FORMAT` 导致 start.sh 报错

根目录 `.env` 中带空格的值需加引号，例如：

```ini
PM2_LOG_DATE_FORMAT="YYYY-MM-DD HH:mm:ss Z"
```

`start.sh` 已避免整文件 `source .env`，但其他工具仍可能受影响。

### 4. API 检测 MCP 时连到 127.0.0.1:31181

本地 HTTP 代理失效时，请：

```bash
unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY
./start.sh restart
```

或永久取消 Git/系统代理。

### 5. ERPNext 连接失败

确认 ERPNext 已启动、URL 与 API Token 正确，且 `8000` 端口可访问。
