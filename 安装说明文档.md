# BuildingAI 本地开发安装说明

本文档说明如何在 **不使用 Docker 运行应用** 的前提下，在本地完成 BuildingAI 的克隆、依赖安装与二次开发环境搭建。适用于 macOS / Linux；Windows 可参考同类工具（nvm、Homebrew 等价物）。

- 项目仓库：[https://github.com/otanjian/BuildingAI](https://github.com/otanjian/BuildingAI)
- 官方文档（Docker 部署）：[部署文档](https://www.buildingai.cc/docs/introduction/install)
- 网址：http://localhost:4091/, Rock, Rock123

---

## 一、环境要求

| 项目 | 要求 |
|------|------|
| CPU | ≥ 2 核 |
| 内存 | ≥ 4 GB |
| 磁盘 | ≥ 5 GB 可用空间 |
| Node.js | `>= 22.20.x` 且 `< 23`（推荐用 nvm 安装 22.x） |
| 包管理器 | pnpm `10.20.0`（见 `package.json` 中 `packageManager`） |
| PostgreSQL | **17.x**（需支持 `pgvector` 扩展） |
| Redis | 6.x / 7.x / 8.x |

---

## 二、获取代码

### 方式 A：Git 克隆（推荐）

```bash
cd /path/to/your/workspace
git clone https://github.com/otanjian/BuildingAI.git buildingAI
cd buildingAI
```

若遇代理导致 `git clone` 失败，可临时取消 Git 代理后再试：

```bash
git config --global --unset http.https://github.com/.proxy
unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY
git clone https://github.com/otanjian/BuildingAI.git buildingAI
```

### 方式 B：下载 ZIP

从 GitHub 下载 `master` 分支 ZIP，解压到目标目录后进入项目根目录。

## 三、安装 Node.js 与 pnpm

### 1. 安装 Node.js 22

```bash
# 使用 nvm（推荐）
nvm install 22
nvm use 22
node -v   # 应显示 v22.x.x
```

### 2. 安装 pnpm

```bash
npm install -g pnpm@10.20.0
pnpm -v
```

若 Corepack 报 SSL 证书错误，可直接使用全局安装的 pnpm，或配置 registry：

```bash
pnpm config set registry https://registry.npmmirror.com
```

---

## 四、安装 PostgreSQL 与 Redis

### macOS（Homebrew）

```bash
# PostgreSQL 17（pgvector 官方包依赖 PG 17/18）
brew install postgresql@17 pgvector redis

brew services start postgresql@17
brew services start redis
```

创建数据库并启用向量扩展：

```bash
# 将 <用户名> 替换为当前系统用户（macOS 下常为 whoami 输出，如 jiantan）
createdb -U <用户名> buildingai
psql -U <用户名> -d buildingai -c "CREATE EXTENSION IF NOT EXISTS vector;"
```

> **说明**：Homebrew 安装的 `pgvector` 面向 PostgreSQL 17/18，请勿仅安装 `postgresql@15` 而未安装 vector 扩展。

### Linux

请安装 PostgreSQL 17，并自行编译或安装对应版本的 [pgvector](https://github.com/pgvector/pgvector)。

## 五、配置环境变量

```bash
cd buildingAI
cp .env.example .env
pnpm sync-env
```

编辑根目录 `.env`，本地开发常用配置示例（**macOS + Homebrew PostgreSQL**）：

```ini
# 服务
SERVER_PORT=4090

# 数据库（Homebrew 默认使用当前系统用户，无密码）
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=<你的系统用户名>
DB_PASSWORD=
DB_DATABASE=buildingai

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# 前端开发时 API 地址
VITE_DEVELOP_APP_BASE_URL=http://localhost:4090
```

> Linux 若使用 `postgres` 用户，可保持 `.env.example` 中的 `DB_USERNAME=postgres` 与 `DB_PASSWORD=postgres`。

## 六、安装依赖与构建

在项目根目录执行（务必使用 Node 22）：

```bash
nvm use 22
pnpm install
```

首次或依赖变更后，建议构建各子包（Monorepo）：

```bash
pnpm build
```

生成扩展配置（首次需要）：

```bash
node packages/cli/bin/predeploy.js
```

或在构建流程中由脚本生成 `extensions/extensions.json`。若启动报错找不到 `extensions.json`，可手动执行上述命令（会扫描 `extensions/` 目录下的 `manifest.json`）。

## 七、启动开发服务

```bash
nvm use 22
pnpm dev
```

| 服务 | 地址 |
|------|------|
| API 后端 | http://localhost:4090 |
| 前端 (Vite) | http://localhost:4091 |

仅启动部分服务：

```bash
pnpm dev:api   # 仅 API
pnpm dev:web   # 仅前端
```

生产/守护进程方式（可选）：

```bash
pnpm start     # 使用 PM2 启动（需先完成安装向导）
```

## 九、常见问题

### 1. `role "postgres" does not exist`

macOS Homebrew PostgreSQL 默认没有 `postgres` 角色。将 `.env` 中 `DB_USERNAME` 改为当前系统用户名，并清空 `DB_PASSWORD`。

### 2. `extension "vector" is not available`

需使用 **PostgreSQL 17+** 并安装 `pgvector`，在数据库中执行：

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### 3. `extension "zhparser" is not available`

`zhparser` 用于知识库中文全文检索。macOS 上原生安装较麻烦；当前代码在初始化失败时会跳过该扩展，**不影响主站与向量检索启动**。

若需完整中文分词，可选：

- 使用官方 Docker 镜像中的 PostgreSQL（仅数据库容器）；或  
- 在 Linux 上自行编译安装 [zhparser](https://github.com/amutu/zhparser)。

### 4. `Unable to find extensions.json`

执行：

```bash
node packages/cli/bin/predeploy.js
```

确保存在 `extensions/extensions.json`。

### 5. 端口被占用

```bash
lsof -ti:4090,4091 | xargs kill -9
pnpm dev
```

### 6. Node 版本不符

项目要求 Node `>=22.20.x <23`，请执行 `nvm use 22` 后再运行 `pnpm` 命令。

### 7. 网络 / 代理问题

- Git：检查 `~/.gitconfig` 是否配置了失效的 `http.proxy`。  
- npm/pnpm：可设置 `registry.npmmirror.com` 或关闭错误的本地代理（如 `127.0.0.1:31181`）。

## 十、目录结构（简要）

```
buildingAI/
├── packages/
│   ├── api/          # NestJS 后端
│   ├── client/       # Vite + React 前端
│   └── @buildingai/  # 共享库
├── extensions/       # 插件（含 extensions.json）
├── .env              # 环境变量（勿提交密钥）
├── package.json
└── pnpm-workspace.yaml
```

## 十一、与 Docker 部署的区别

| 方式 | 适用场景 |
|------|----------|
| **本文档（本地源码）** | 二次开发、调试、改代码 |
| **Docker Compose** | 快速体验、生产部署，见 `README.zh-CN.md` |

本地开发时应用进程在宿主机运行，PostgreSQL / Redis 可在本机安装，也可仅将数据库放入 Docker（需自行配置 `.env` 连接信息）。
