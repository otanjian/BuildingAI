3. 核心需求：MRP数据治理
通过mcp 从ERP系统获取mrp相关的数据，

4. 功能需求

4.1 数据采集（MCP 资源）
通过mcp从 ERP 获取以下数据集。。

资源逻辑组	资源描述	数据内容	建议更新频率
mrp.materials	物料主数据	物料编码、名称、提前期、安全库存、订货策略、状态（启用/停用）
mrp.bom	物料清单	BOM 头（父项、版本、生效/失效时间）及行（子项、用量、损耗率）
mrp.inventory	当前库存快照	物料、仓库、库存状态（合格/待检/冻结）、可用量	每小时
mrp.inventory_moves	库存移动单据	近 7 天所有入库/出库/移库/发料单据（含时间、数量、关联订单号）	每小时增量
mrp.quality_records	质量检验记录	近 7 天检验单、拒收单、让步接收单（含物料、数量、判定结果）	每小时
mrp.sales_orders	未结销售订单	订单号、物料、数量、需求日期、状态	每小时
mrp.purchase_orders	未结采购订单	订单号、物料、数量、计划到货日、确认到货日、已收量	每小时
mrp.production_orders	未结生产订单	订单号、物料、计划数量、合格入库量、计划开始/完工日、实际报工数据	每小时
mrp.planned_orders	MRP 计划订单	计划订单号、物料、数量、建议日期、确认状态	每次 MRP 运行后
4.2 数据质量检测规则（AI + 规则引擎）
AI 智能体对采集到的数据执行以下检测，输出 问题清单（严重程度：高/中/低）。

4.2.1 静态数据检测
检测项	规则/方法	严重程度
物料提前期为 0 或负数	直接过滤	高
提前期显著偏离历史实际（如系统 30 天，近 3 次采购平均 45 天）	统计学异常检测（Z-score）	中
安全库存设置不合理（导致连续缺货或积压）	计算需求波动与服务水平	中
BOM 用量与实际生产消耗偏差 >20%	对比生产订单发料记录	高
BOM 版本过期（物料已停用仍在 BOM 中）	关联物料状态	高
同一物料存在多编码（一物多码）	文本相似度 + 规格聚类	中
4.2.2 动态交易数据检测
检测项	规则/方法	严重程度
库存移动单据未及时过账（如出库时间 > 实际发货时间 12 小时）	对比 WMS 时间戳（若有）或单据创建时间与操作时间	高
账面库存与循环盘点差异 >5%	比对 ERP 库存与盘点导入数据	高
采购订单“确认到货日”为空或已过期未更新	检查字段空值 + 当前日期>计划到货日	高
采购订单欠交数量未处理（已收量 < 订单量，且无新 PO）	关联采购订单与入库单	中
生产订单未报工但已完工入库	生产订单状态不一致	高
质检完成但库存状态仍为“待检”超过 2 小时	检验单完成时间 vs 库存状态变更时间	高
4.2.3 订单执行偏差检测
检测项	规则/方法	严重程度
销售订单交期变更未同步到系统	比对 CRM/邮件与 ERP 订单的交期差异（初版可手工标记）	中
采购订单实际到货日 > 确认到货日 3 天以上	简单阈值	高
生产订单实际完工日 > 计划完工日，且无延期申请	延期天数阈值	高
销售订单已关闭但仍有在途采购/生产	关联需求-供应	高
4.3 智能诊断
对于检测到的问题，AI 智能体需进行根因分析，输出自然语言解释。

示例：

问题：物料 M-1024 账面库存 1000，实物盘点 700，差异率 30%。
AI 诊断：检查近 3 天库存移动单据，发现 2 笔生产发料单（PRD-001, PRD-002）在系统创建时间早于实际发料时间 24 小时，且发料数量虚高。建议责任人追查该工单是否有退料未入账。

技术实现：AI Agent 可调用 MCP 工具获取关联单据详情，结合时序数据和规则模式匹配（或大模型推理）给出结论。

4.4 治理建议与自动更新（通过 MCP 工具调用）
针对诊断结果，AI 生成治理建议。AI 不预置任何具体工具名称，而是：

通过 MCP 协议的 tools/list 接口获取 ERP 侧当前可用的所有工具列表。

理解每个工具的描述、输入参数 Schema、输出格式。

根据治理意图（如“更新物料提前期”、“关闭采购订单”），选择合适的工具并构造参数。

调用工具，并处理返回结果。

若 ERP 未提供完成某治理动作所需的工具，AI 应生成人工操作指引。

典型治理动作类型（示例，不代表具体工具）：

治理动作类型	描述	所需工具语义（示例）
更新物料提前期	根据历史实际采购周期修正	一个能够修改物料主数据中“提前期”字段的工具
调整安全库存	根据需求波动重新计算	一个能够修改物料主数据中“安全库存”字段的工具
关闭过期采购订单	关闭长期未交货且不再需要的 PO	一个能够将采购订单状态置为“关闭”或“取消”的工具
调整 BOM 用量	修正 BOM 子项用量与损耗率	一个能够修改 BOM 明细行中“用量”或“损耗率”的工具
触发库存盘点	对差异物料生成盘点任务	一个能够创建盘点单据或盘点任务的工具
同步销售订单交期	根据最新客户确认更新需求日期	一个能够修改销售订单中“需求日期”字段的工具
重算计划订单	清理无效计划订单，触发 MRP 重算	一个能够触发 MRP 运算或清理计划订单的工具
安全机制：

所有通过 MCP 工具执行的写操作，AI 需在调用前生成操作说明（含预期影响），推送到看板。

关键操作（如修改 BOM、调整安全库存）必须经过人工审批，审批通过后 AI 再调用工具。

非关键操作（如创建盘点任务）可配置为自动执行。

4.5 治理看板与告警
提供一个 Web 仪表盘（可嵌入现有系统），展示：

数据质量总体评分（0-100）

各维度（静态/库存/订单）问题数量与趋势

待审批的 AI 建议列表（含详细理由、拟调用的工具和参数、模拟更新前后对比）

已执行的操作历史（含调用的工具名称、参数、结果）

配置页面：检测规则阈值、自动执行白名单（哪些工具可以自动调用）、审批人设置

同时支持邮件/企业微信告警，对高严重程度问题实时通知数据治理专员。

5. MCP 接口约定（供 ERP 实现方参考）
ERP 系统需实现一个 MCP 服务端，对外暴露必要的资源和工具。本 PRD 不对具体资源 URI、工具名称做强制规定，但提出以下建议约定，以便 AI 能更好地理解和使用。

5.1 资源建议
资源应提供以下逻辑数据集（命名和路径自由）：

物料主数据

BOM 结构

当前库存快照

库存移动单据

质量检验记录

未结销售订单

未结采购订单

未结生产订单

MRP 计划订单

每个资源应支持分页、过滤（按时间、物料等）和增量拉取（如通过 updated_after 参数）。

5.2 工具建议
ERP 应根据自身能力暴露一组工具。工具应满足：

工具名称清晰反映其功能（如 update_item_lead_time、close_purchase_order、create_inventory_count_task 等）。

工具描述（description）应详细说明用途、适用条件、副作用。

输入参数使用 JSON Schema 明确定义，包含参数名称、类型、是否必填、取值范围、示例。

输出格式统一（包含 success 布尔值、message 字符串、必要时返回 data 对象）。

不强制要求实现某一套固定工具集。AI 会通过 tools/list 动态发现可用工具，并在缺少必要工具时生成人工操作指引。
